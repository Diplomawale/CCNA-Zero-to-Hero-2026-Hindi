# 📚 What is OSI Model?

OSI (Open Systems Interconnection) Model is a conceptual framework used to understand how data travels from one device to another across a network.

It divides network communication into **7 layers**, making networking easier to understand, design, and troubleshoot.

---

# 🎯 Why OSI Model is Important?

✅ Standardizes communication

✅ Simplifies troubleshooting

✅ Helps understand data flow

✅ Foundation for CCNA and Networking

✅ Used by Network Engineers worldwide

---

# 🏗️ OSI 7 Layers

| Layer | Name         | Function                 |
| ----- | ------------ | ------------------------ |
| 7     | Application  | User interaction         |
| 6     | Presentation | Encryption & Compression |
| 5     | Session      | Session Management       |
| 4     | Transport    | TCP/UDP Communication    |
| 3     | Network      | IP Address & Routing     |
| 2     | Data Link    | MAC Address & Switching  |
| 1     | Physical     | Cables & Signals         |

---

# 🔄 Data Flow Example

User opens YouTube on mobile:

```text
Application Layer
        ↓
Presentation Layer
        ↓
Session Layer
        ↓
Transport Layer
        ↓
Network Layer
        ↓
Data Link Layer
        ↓
Physical Layer
        ↓
Switch → Router → ISP → Internet → YouTube Server
```

# 🔍 Layer-wise Explanation

## Layer 7 — Application Layer

Examples:

* YouTube
* Chrome Browser
* WhatsApp
* Email

Purpose:

Provides network services to end users.

---

## Layer 6 — Presentation Layer

Functions:

* Encryption
* Compression
* Data Translation

Example:

HTTPS encryption.

---

## Layer 5 — Session Layer

Functions:

* Session Establishment
* Session Maintenance
* Session Termination

Example:

Login session between client and server.

---

## Layer 4 — Transport Layer

Protocols:

* TCP
* UDP

Functions:

* Segmentation
* Reliability
* Flow Control

---

## Layer 3 — Network Layer

Functions:

* Routing
* Logical Addressing

Device:

* Router

Address:

* IP Address

Example:

192.168.1.10

---

## Layer 2 — Data Link Layer

Functions:

* Frame Creation
* MAC Address Communication

Device:

* Switch

Address:

* MAC Address

---

## Layer 1 — Physical Layer

Functions:

* Signal Transmission
* Cable Communication

Examples:

* Ethernet Cable
* Fiber Cable
* RF Link

---

# 🛠️ Troubleshooting Using OSI Model

### Layer 1 Issue

❌ Cable disconnected

### Layer 2 Issue

❌ MAC Learning Problem

### Layer 3 Issue

❌ IP Address Incorrect

### Layer 4 Issue

❌ TCP Port Blocked

### Layer 7 Issue

❌ Application Not Working
